## 1. 背景与战略（"为什么"）

### 1.1 问题陈述

养宠人群在日常生活中经常拍摄宠物的各种表情和动作，却难以用文字精准表达宠物"此刻在想什么"。现有的图片编辑工具只能添加固定文案，缺乏个性化和趣味性，无法满足用户在社交平台分享时追求独特内容的需求。

> **核心痛点：** 你家猫现在盯着墙角发呆，它到底在想什么？用户渴望赋予宠物"人格"，但缺乏创作能力或灵感。

### 1.2 机会与目标

* **主要目标：** 打造一款基于多模态 AI 的宠物情绪解码工具，让每张宠物照片都能生成有趣、个性化的"内心独白"
* **次要目标：**
  * 建立养宠社群的社交裂变传播机制
  * 探索宠物行业品牌合作的商业化路径

### 1.3 成功指标（KPI）

* [ ] **用户增长：** 上线 3 个月内 DAU 达到 10,000
* [ ] **内容生成：** 日均生成内心戏 50,000 条
* [ ] **传播效果：** 分享率 > 30%（生成内容被分享到社交平台的比例）
* [ ] **用户留存：** 7 日留存率 > 25%
* [ ] **付费转化：** 付费用户占比 > 5%

## 2. 目标用户（"谁"）

### 2.1 目标受众

* **主要用户画像：**
  * **年轻养宠族（18-35岁）**：活跃于社交媒体，喜欢分享宠物日常，追求有趣、个性化的内容表达
  * 典型特征：养猫/养狗，每天拍摄大量宠物照片，是小红书/朋友圈/微博的重度用户

* **次要用户画像：**
  * **云养宠群体**：虽未养宠物，但对萌宠内容有强烈兴趣，愿意为朋友的宠物照片生成内心戏
  * **宠物博主/KOL**：需要持续产出有趣宠物内容的创作者

### 2.2 用户故事

| ID | 作为... | 我想要... | 以便... | 优先级 |
| --- | --- | --- | --- | --- |
| **US.1** | 养猫人 | 上传一张猫咪翻白眼的照片 | 让 AI 帮我写出它此刻的内心独白 | **P0（必须）** |
| **US.2** | 养宠人 | 为我的宠物选择一个性格人设（如高冷、傲娇） | 生成的内心戏更符合它的"人设" | **P0（必须）** |
| **US.3** | 社交达人 | 一键生成带文字的表情包图片 | 直接分享到朋友圈/小红书 | **P0（必须）** |
| **US.4** | 养宠人 | 建立我宠物的专属档案（名字、品种、性格标签） | 每次生成的内容更加个性化和连贯 | **P1（应该）** |
| **US.5** | 养宠人 | 生成"宠物朋友圈"样式的图片 | 获得更有趣的社交分享形式 | **P1（应该）** |
| **US.6** | 养宠人 | 偶尔收到"主子写给我的信" | 增加与宠物的情感连接 | **P2（锦上添花）** |

## 3. 解决方案（"是什么"）

### 3.1 功能需求

#### 3.1.1 快拍解码（Snapshot）— 核心功能

| ID | 需求描述 | 验收标准 | 优先级 |
| --- | --- | --- | --- |
| **FR.1.1** | 照片上传 | 支持从相册选择或实时拍摄，支持 JPG/PNG/HEIC 格式，单张最大 10MB | **P0** |
| **FR.1.2** | AI 图像识别 | 调用多模态 API（GPT-4o/Claude）识别：宠物品种、表情、动作、环境背景、物体 | **P0** |
| **FR.1.3** | 内心戏生成 | 基于图像识别结果 + 选定人设，生成 3 个不同版本的内心独白供用户选择 | **P0** |
| **FR.1.4** | 生成结果展示 | 在图片下方展示生成的内心戏文案，支持左右滑动切换版本 | **P0** |
| **FR.1.5** | 重新生成 | 用户对结果不满意时，可点击"换一批"重新生成 | **P1** |

#### 3.1.2 性格实验室（Persona）

| ID | 需求描述 | 验收标准 | 优先级 |
| --- | --- | --- | --- |
| **FR.2.1** | 预设人设模板 | 提供至少 6 种基础人设：高冷总裁、碎碎念大妈、文艺青年、热血少年、毒舌吐槽、卑微打工人 | **P0** |
| **FR.2.2** | 人设选择器 | 上传照片后可选择人设，默认使用"高冷主子" | **P0** |
| **FR.2.3** | 高级人设（付费） | 提供特殊人设：甄嬛体、赛博朋克、霸总文学、古风诗词等，需订阅解锁 | **P1** |

#### 3.1.3 一键梗图（Meme Maker）

| ID | 需求描述 | 验收标准 | 优先级 |
| --- | --- | --- | --- |
| **FR.3.1** | 梗图生成 | 将选定的内心戏文案自动排版在图片上 | **P0** |
| **FR.3.2** | 字体选择 | 提供至少 5 种网红字体样式 | **P1** |
| **FR.3.3** | 滤镜效果 | 提供至少 3 种图片滤镜（原图、复古、黑白） | **P1** |
| **FR.3.4** | 水印/Logo | 生成的图片带有产品水印，付费用户可去除 | **P0** |

#### 3.1.4 社交分享

| ID | 需求描述 | 验收标准 | 优先级 |
| --- | --- | --- | --- |
| **FR.4.1** | 一键分享 | 支持分享到微信好友、朋友圈、小红书、微博 | **P0** |
| **FR.4.2** | 图片保存 | 支持保存到本地相册 | **P0** |
| **FR.4.3** | 裂变二维码 | 生成的图片带有小程序码/二维码，好友扫码可直接使用 | **P1** |

#### 3.1.5 宠物档案（Pet Profile）

| ID | 需求描述 | 验收标准 | 优先级 |
| --- | --- | --- | --- |
| **FR.5.1** | 创建档案 | 用户可为宠物建立档案：名字、品种、性别、年龄 | **P1** |
| **FR.5.2** | 性格标签 | 支持选择/自定义性格标签：拆家狂、社恐、小公举、话痨等 | **P1** |
| **FR.5.3** | 档案关联 | 上传照片时可选择关联已有宠物档案，生成更个性化内容 | **P1** |

### 3.2 边界情况与错误处理

| 场景 | 预期行为 | 优先级 |
| --- | --- | --- |
| 上传的图片中没有识别到宠物 | 提示"未检测到宠物，请上传包含宠物的照片"，并建议重新选择 | **P0** |
| 图片过大或格式不支持 | 提示具体原因，引导用户压缩或更换格式 | **P0** |
| API 调用失败/超时 | 显示友好错误提示，提供"重试"按钮 | **P0** |
| 用户网络断开 | 缓存用户操作，网络恢复后继续 | **P1** |
| 生成内容包含不当言论 | 内容过滤机制，替换或重新生成 | **P0** |
| 免费用户达到每日使用上限 | 提示已达上限，引导开通会员或观看广告 | **P0** |
| 识别到多只宠物 | 询问用户选择主角，或生成对话模式内容 | **P1** |

### 3.3 技术与非功能需求

* **性能：**
  * 图片上传完成后 3 秒内返回生成结果
  * 梗图生成 1 秒内完成
  * 支持 10,000 并发用户

* **安全：**
  * 用户上传的图片仅用于即时处理，处理完成后 24 小时内自动删除
  * 不存储用户隐私照片（除非用户主动保存到档案）
  * API 密钥加密存储，不暴露给前端

* **内容安全：**
  * 生成内容需经过敏感词过滤
  * 上传图片需进行合规审核（排除违规内容）

* **无障碍：** 暂不在 MVP 范围内

* **本地化：**
  * 第一阶段仅支持简体中文
  * 第二阶段考虑繁体中文、英语

* **平台支持：**
  * 第一阶段：PWA 网页版

## 4. 发布与执行（"如何"）

### 4.1 范围与阶段划分

| 阶段 | 范围 | 目标日期 |
| --- | --- | --- |
| **第一阶段（MVP）** | 快拍解码（核心）、6 种基础人设、梗图生成（基础版）、社交分享、免费用户每日 5 次限制 | 待定 |
| **第二阶段（增长版）** | 宠物档案、高级人设（付费）、去水印（付费）、多字体/滤镜、裂变二维码 | 待定 |
| **第三阶段（完整版）** | 主子朋友圈、宠物信件、对话模式、品牌合作植入 | 待定 |

**不在范围内：**
* 实体周边（如定制手机壳、T恤）— 需要供应链能力，暂不考虑
* 视频内容生成 — 技术复杂度高，暂不支持
* 多语言支持 — 第一阶段仅中文

### 4.2 发布策略

* **灰度发布：**
  * 第 1 周：内测用户 500 人
  * 第 2 周：扩大至 5,000 人
  * 第 3 周：全量开放
* **回滚触发条件：**
  * API 错误率 > 10%
  * 用户投诉率 > 5%
  * 发现内容安全问题
* **回滚方案：** 关闭小程序入口，发布修复版本

### 4.3 依赖项

* [ ] **多模态 API 接入：** 需要接入 GPT-4o 或 Claude API（状态：待评估）
* [ ] **图片处理服务：** 需要云端图片处理能力（压缩、裁剪、合成）
* [ ] **微信小程序开发资质：** 需要完成小程序注册和审核
* [ ] **内容安全服务：** 接入第三方内容审核 API 或使用微信安全接口
* [ ] **云服务器资源：** 预估需要的计算和存储资源

### 4.4 测试要求

* **单元测试：** 覆盖核心业务逻辑（图片处理、文案生成、人设匹配）
* **集成测试：** API 调用链路完整性测试
* **端到端测试：** 完整用户流程（上传 → 生成 → 分享）
* **UAT 标准：** 内测用户完成所有 P0 功能体验，满意度 > 80%
* **性能测试：** 模拟 1,000 并发用户压力测试
* **兼容性测试：** 覆盖主流微信版本和机型

* *问：生成内容的版权归属如何界定？*
* *答：待法务确认*

### 5.2 已知风险

| 风险 | 影响 | 概率 | 缓解措施 |
| --- | --- | --- | --- |
| API 调用成本过高 | 高流量下服务器成本失控 | 中 | 设置用户使用频率限制；缓存相似请求；优化 Prompt 减少 token 消耗 |
| 用户"玩过即走"留存率低 | 难以建立可持续业务 | 高 | 引入宠物档案增强个性化；推送"宠物信件"增加回访动机 |
| 生成内容质量不稳定 | 影响用户体验和口碑 | 中 | 提供多版本选择；允许重新生成；持续优化 Prompt |
| 内容安全风险 | 可能生成不当言论 | 低 | 接入内容过滤；人工审核机制 |
| 竞品快速跟进 | 市场先发优势丧失 | 中 | 快速迭代；建立用户宠物数据壁垒 |

### 5.3 决策记录

* *日期：2026-01-19* — 初版 PRD 创建，确定 MVP 范围为核心生成功能 + 基础分享能力。

## 6. 商业模式

### 6.1 收入来源

| 收入类型 | 描述 | 阶段 |
| --- | --- | --- |
| **订阅会员** | 解锁高级人设、去水印、无限次使用 | 第二阶段 |
| **激励广告** | 免费用户观看广告获得额外次数 | 第二阶段 |
| **品牌植入** | 内心戏中自然植入宠物品牌（如罐头、猫砂） | 第三阶段 |
| **B端合作** | 为宠物品牌提供定制化内容生成服务 | 第三阶段 |

### 6.2 定价策略（建议）

* **免费版：** 每日 5 次生成，带水印，6 种基础人设
* **月度会员：** ¥12/月，无限次数，去水印，全部人设
* **年度会员：** ¥98/年，所有会员权益 + 专属人设更新
